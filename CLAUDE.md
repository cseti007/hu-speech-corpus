# Project guide for new Claude sessions

Read this file FIRST before doing anything else in this project. It tells you
where things live and what operating rules this project has.

## What this project is

Building a free-license Hungarian speech corpus for ASR/TTS LoRA training.
~22,820 hours of HU audio downloaded (mostly EP parliamentary, some YouTube +
audiobooks + podcasts). Phase 1 (data acquisition) is complete; we are now in
Phase 2 (segmentation, manifest refinement, training pipeline decisions).

## Operating rules — IMPORTANT

### Rule 1: STATS.md is the canonical source of truth for "how big is the dataset"

The user's most-asked question is **"how many hours of audio do we have right now?"**.
That answer lives in `STATS.md` and must always be current.

Whenever you do any of these things, update `STATS.md`:
- Download a new source (or finish one)
- Run segmentation / preprocessing that changes effective hours
- Filter / re-label rows in manifests
- Re-run `bin/build_manifest.py` (the output `processed/manifests/stats.json` is the source data; STATS.md is the human-readable summary)
- Delete or re-encode any source

The `processed/manifests/stats.json` file is auto-generated by `build_manifest.py`
and contains the raw numbers. `STATS.md` is the human-curated version with
context.

### Rule 2: Update CHECKLIST.md as you go

When you complete a pending item, mark it `[x]` with a short result note.
When you discover a new task, add it under the right section.
Don't leave stale items.

### Rule 3: Plan twice, cut once (per global CLAUDE.md)

This project follows the user's global rules:
- Hungarian for discussion, English for code/files/commits
- Ask before creating, modifying, or deleting files unless durably authorized
- No emojis in files
- Surgical edits — don't refactor adjacent code beyond what's asked

### Rule 4: HF token

The Hugging Face token is at `/home/cseti/.hf_token` (read-only file).
Scripts read it via `load_token()` helper. Do not commit it, do not print it.

### Rule 5: Conda env

This project uses base conda:
`/media/cseti/datassd/conda/miniconda3/bin/python`

System `/usr/bin/python3` lacks the required libraries (`huggingface_hub`, `pyarrow`,
`pyyaml`, `tqdm`). Always invoke conda Python explicitly.

### Rule 6: Idempotent everything

Every script must be idempotent (safe to re-run; skip already-done work).
For HF downloads: `snapshot_download` handles this. For custom downloads:
maintain a `progress.json`. For pipeline steps: check sentinel files
(`.download_complete`, `.segmentation_complete`, etc.).

## First-session checklist for a new Claude

When a user opens a fresh session in this directory:

1. Read `STATS.md` to know current corpus size — answer "how many hours" instantly.
2. Read `CHECKLIST.md` to know what's pending.
3. Read `STATUS.md` if you need the prose-level project state and recent decisions.
4. Read `configs/sources.yaml` if you need per-source detail.
5. Only then start whatever the user asked for.

Skip the rest of `notes/` unless the user references a specific topic.

## How to refresh stats after a state change

```bash
cd /home/cseti/data2/Develop/Github-cseti/cseti-os/projects/hu-speech-corpus
/media/cseti/datassd/conda/miniconda3/bin/python -u bin/build_manifest.py
# then update STATS.md from the new processed/manifests/stats.json
```

Manifest rebuild on full corpus (~30 min due to 17k ffprobe calls):
- aligned: ~3 min
- unaligned: ~25-30 min (ffprobe on VoxPopuli sessions)
- pseudo_labeled: ~5 min (MOSEL TSV CSV parse)

Pass `--skip-mosel` to skip the largest file when only aligned/unaligned matters.
